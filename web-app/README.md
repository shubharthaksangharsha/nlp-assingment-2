# NLP Knowledge Base

A web application designed to explore and visualize a knowledge base of Natural Language Processing (NLP) questions and insights gathered from the Stack Exchange network.

by [Shubharthak Sangharasha](https://shubharthaksangharsha.github.io/)

## Description 

This project fetches, processes, categorizes, and displays NLP-related questions and answers. It provides users with an interface to browse questions by category, view detailed posts, search the knowledge base, and explore data visualizations derived from the dataset. The application is built with Flask and designed for deployment on Vercel.

[DEMO](https://nlp-assingment2-hwg887pak-shubharthaks-projects.vercel.app/)
## Features

- **Browse Categories:** Explore questions organized into various NLP categories (e.g., Tasks, Libraries, Question Types).
- **View Posts:** See the details of individual questions, including descriptions and answers.
- **Visualizations:** View charts and graphs summarizing trends and patterns in the NLP data (e.g., top tags, word clouds).
- **Search:** (Functionality might exist or be planned) Search through the knowledge base.
- **API Endpoints:** Provides basic API access to categories and dataset statistics.
- **Admin Functions:** Includes an endpoint to reprocess categories (requires source data).
- **Vercel Ready:** Configured for easy deployment via Vercel.

## Tech Stack

- **Backend:** Python 3
- **Framework:** Flask
- **Data Processing:** Pandas
- **Frontend:** HTML, CSS, JavaScript, Bootstrap 5
- **Visualizations (Source):** Matplotlib, Seaborn (used to generate static images)
- **Deployment:** Vercel, Gunicorn (as WSGI server)

## Setup and Local Development

1. **Clone the repository:**
    ```bash
    git clone <your-repository-url>
    cd <repository-directory>
    ```

2. **Create a virtual environment (recommended):**
    ```bash
    python3 -m venv venv
    source venv/bin/activate  # On Windows use `venv\Scripts\activate`
    ```

3. **Install dependencies:**
    ```bash
    pip install -r requirements.txt
    ```

4. **Data:**
    - Ensure the `data` directory with its `categories` subdirectories and files is present in the project root. This data is generated by separate processing scripts (not included in the runtime app).


5. **Environment Variables:**
    - The application uses `STACK_API_KEY` (optional, used for certain potential API interactions or future features, defaults provided). You might need to set this if required by specific functionality. For local development, you can set it in your terminal:
        ```bash
        export STACK_API_KEY='your_actual_key' # Linux/macOS
        # set STACK_API_KEY=your_actual_key    # Windows Command Prompt
        # $env:STACK_API_KEY='your_actual_key' # Windows PowerShell
        ```

6. **Run the application:**
    ```bash
    python app.py
    ```
    Or use the provided run scripts:
    ```bash
    ./run_webapp.sh  # Linux/macOS (make sure it's executable: chmod +x run_webapp.sh)
    .\run_webapp.bat # Windows
    ```
    The application will be available at `http://127.0.0.1:5000` or `http://0.0.0.0:5000`.

## Deployment (Vercel)

This application is pre-configured for deployment on Vercel.

1. **Prerequisites:**
    - Ensure `vercel.json` is present in the root directory.
    - Ensure `requirements.txt` lists all dependencies, including `gunicorn`.
    - Ensure the `data` directory (with category files) is included in your Git repository and located in the project root.
    - Ensure all required visualization images are located within the `static/img` directory and committed to Git. The `vercel_setup.py` script (run by `app.py` on Vercel) currently *skips* the automatic copying of visualizations, assuming they are already in `static/img`.

2. **Connect & Deploy:**
    - Push your repository to GitHub/GitLab/Bitbucket.
    - Import the repository into Vercel.
    - Vercel should automatically detect the Python runtime and use `vercel.json` for build and routing configuration.
    - **Environment Variables:** Set any required environment variables (like `STACK_API_KEY`) in the Vercel project settings.
    - Click "Deploy".

## API Endpoints

- `/api/categories`: Returns a JSON object of all available category types and their associated categories (name, count, file). (See `app.py`)
- `/api/stats`: Returns basic statistics about the dataset. (See `app.py`)

## Admin

- `/admin/reprocess_categories`: An endpoint to trigger reprocessing of categories. Requires the source `preprocessed_nlp_dataset.csv` to be available in the `data` directory and the `src.categorizer` module to be present. Use with caution. (See `app.py`)
